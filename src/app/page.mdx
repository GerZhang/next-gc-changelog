## 提供 MCP 支持，让 AI 智能体连接万物 {{ type: 'feature', typeText: 'AI' }}

就像 USB-C 为电子设备提供了一种标准化的连接方式一样，MCP（模型上下文协议）是一个用于连接 AI 应用程序和外部系统的开源标准。

像 DeepSeek 或 ChatGPT 这样的 AI 应用程序可以通过 MCP 连接到数据源（例如本地文件、数据库）、工具（例如搜索引擎、计算器）和工作流程（例如管理软件能力），从而能够访问关键信息并执行任务。

在 V11 Update 1 版本中，开发者可以在活字格应用中调用第三方的 MCP 服务，也可以将活字格的「服务端命令」构建为 MCP Server，向第三方提供服务。

- **调用 MCP 服务**。开发者可以将活字格当做 MCP Client，调用本地或者第三方的 MCP 服务。例如借助于高德 MCP，为您的客户提供个性化的出行建议和形成规划；借助联网搜索 MCP，让活字格应用具备实时获取外部数据的能力。
- **构建 MCP Server**。开发者可以将活字格的「服务端命令」构建为 MCP Server，对外提供 MCP 服务。如果应用使用设计器本地运行时，该 MCP Server 仅允许活字格设计器本地调用。当应用发布后，在配置鉴权后，允许第三方进行调用。
- 「AI 对话单元格」与「AI 助手命令」均支持 MCP 工具调用。
- 同时，插件开发也提供了 MCP 属性与相关 API 支持。

## 提升智能体开发体验 {{ type: 'optimize', typeText: 'AI' }}

活字格 V11 版本推出的 AI 智能体构建能力，已成为产品的核心能力之一。因此，智能体开发的体验优化也成为了我们的重点关注对象。

- 提示词编辑器现在支持上下文的联想功能。当开发者在编写提示词时，系统会根据输入内容，自动提示关联的上下文变量、Excel 公式、系统关键字等，并在选中后进行标红，帮助开发者更快、更准确地编写提示词。
- 服务端「AI 对话助手命令」现已支持独立测试。开发者可以在开发过程中，单独基于该命令，测试 AI 智能体的功能，无需像历史版本一样测试整个命令树逻辑。

## UI 设计逻辑重构 {{ type: 'feature', typeText: 'UI' }}

当界面工作量较多时，活字格在 UI 设计以及单元格类型配置上会出现大量的重复性工作。因此在 V11 update 1 中，我们对设计逻辑进行了重构（非破坏性更新），以提高开发者的设计体验与开发效率，并为开发者沉淀设计规范提供了产品级的能力支撑。

- 全新的「预置模板管理」模块。我们将设计器中的设计页签彻底重构，上线全新的「预置模板管理」功能，按照复用粒度提供「单元格类型模板」、「区块模板」以及「自定义页面模板」三种自定义模板类型，开发者可以根据自己的需求，创建、管理、使用这些模板，以提高开发效率和设计质量。模板支持单独导入导出，便于开发者进行规范分享。
- 全局主题管理。开发者可以在「主题管理」中，统一整个工程的字体、颜色、单元格样式以及表格样式。
- 优化界面元素创建过程。当开发者在页面中创建元素时，现在支持直接从预置模板中进行快速选择。

## 权限组模式 {{ type: 'feature', typeText: '权限' }}

 过去，活字格仅提供按照角色模式进行权限分配，并没有提供标准的权限维度，需要开发者用角色维度进行权限模拟。这对于开发者上手活字格权限设计，带来了较高的门槛。如今，该问题将会被新引入的「权限组」模式所解决。

- 活字格现已明确提供权限维度，无需开发者自行实现。在权限组的设计模式下，开发者可以直接创建管理权限组，并为其维护相应的具体权限职能与角色关联。系统仍保留传统的「角色模式」以确保老版本应用的兼容。
- 支持自定义前端权限配置。基于全新的权限组模式，我们提供了新的命令插件「角色权限组管理」。借助该插件提供的命令，开发者可以在前端页面中，自定义构建配置和管理权限组的模块页面。

## 支持分级授权 {{ type: 'feature', typeText: '权限' }}

基于新的「权限组」模式。分级授权也被活字格支持。在 V11 Update 1 版本中，服务器管理控制台新增了名为「分级授权」的管理后台权限模式。开启后，企业可以基于实际情况，在应用层面上，对于组织结构中的各个节点做精细化的权限管控。典型支持场景示例：

1. 集团管权限，子部门分管各自的应用。例如公司下 A、B两个部门都可以发布自己部门的专属应用，但彼此都看不到对方。
2. 集团子公司共用应用，但是权限需要基于各个子公司精细化管控。例如公司下 A、B两个部门，应用会被 A、B两个部门的员工共同使用。但是允许 A 部门的管理员管理应用，但是 B 部门的管理员无权管理。

## 开发体验优化 {{ type: 'optimize', typeText: '设计器' }}

增强开发者的开发体验，是活字格每个版本功能迭代必须考虑的重要问题。

- 「图文列表」优化。现已支持配置 JSON 数据源和 Excel 公式，开发者**不再需要设置隐藏表格来辅助处理动态数据**。

  此外，我们还优化了「图文列表」的性能：相较老版本，PC 端的性能提升 <span style={{ color: '#ff6600', fontWeight: 'bold' }}>100% ↑</span>；移动端（PDA）的性能提升<span style={{ color: '#ff6600', fontWeight: 'bold' }}>超过 200% ↑ </span>。
- 「服务端命令」的参数新增 **对象类型**。开发者可以在服务端命令的参数中，配置参数类型为对象，这样在命令上下文中就可以直接获得对象字段。
- 命令注释增强。为了更好地支持开发者在使用命令时的理解和调试，我们增强了命令注释的功能。对于原先的注释功能，从悬浮提示改为了传统编码后缀注释风格的显式呈现。

  提供命令别名功能，允许开发者为命令提供新的别名，以实现自然语言描述伪代码的方式呈现命令逻辑，如标题图所示。

除上述重点更新外，还增加了许多开发细节的优化：
- 命令树增加辅助线，方便开发者更好的理解命令逻辑之间的层级关系。
- 部分内置命令增加返回值。开发者可以借助于新增的返回值，对内置命令的执行结果，定制相应的处理逻辑，例如「导入 excel 数据到表格」、「导入导出 CSV 命令」等。
- 「设置变量命令」支持获取子表的数组数据，自动构建完整对象。
