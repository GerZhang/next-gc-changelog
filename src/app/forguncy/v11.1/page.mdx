## 提供 MCP 支持，让 AI 应用连接万物 {{ type: 'feature', typeText: 'AI' }}

就像 USB-C 让各种电子设备轻松互联一样，**MCP（Model Context Protocol，模型上下文协议）** 正在成为 AI 应用与外部世界对话的“通用接口”。

通过 MCP，像 DeepSeek、ChatGPT 这样的 AI 应用程序可以安全、标准化地连接到数据源（如数据库、本地文件）、工具（如搜索引擎、计算器）甚至业务系统（如 ERP、CRM），从而真正“看得见、做得了”。

在 V11 Update 1 中，活字格全面接入 MCP 生态，让你的应用既能**调用外部 AI 能力**，也能**对外提供服务**，成为 AI 应用生态中的一员：

### 🔌 作为 MCP 客户端：轻松调用外部智能服务

你可以将活字格应用配置为 MCP Client，直接调用本地或第三方提供的 MCP 服务。例如：

- 接入 **高德地图 MCP**，为用户提供个性化出行建议与路线规划；
- 使用 **联网搜索 MCP**，让 AI 助手实时获取最新资讯或市场数据；
- 连接 **企业内部工具 MCP**，让 AI 自动查询库存、审批状态等业务信息。

### 🚀 作为 MCP 服务端：把你的业务能力开放给 AI

你还可以将活字格中的 「服务端命令」发布为 MCP Server，让外部 AI 应用安全调用你的业务逻辑：

- 在设计器本地运行时，MCP Server 仅限本地调试使用；
- 应用发布后，配合权限与鉴权配置，即可向外部 AI 系统开放服务，比如让智能客服自动调用你的订单查询或用户管理接口。

### 💬 AI 功能全面打通 MCP

- 「AI 对话单元格」 和 「AI 助手命令」 现已原生支持 MCP 工具调用，AI 不再只是“聊天”，而是能真正执行任务的智能体。
- 同时，插件开发也开放了 MCP 相关属性与 API，方便开发者深度集成自定义 MCP 能力。

通过 MCP 支持，All in One 的活字格开发平台，已经正式成为 AI 与企业系统之间的“智能桥梁”。无论你是想让 AI 更懂业务，还是让业务系统更智能，现在都有了标准、安全、高效的实现路径。

## 全新「权限组」模式上线 {{ type: 'feature', typeText: '权限' }}

过去，活字格的权限体系仅基于“角色”进行分配，缺乏明确的权限维度。开发者往往需要通过复杂的角色组合来“模拟”实际权限需求，不仅上手门槛高，也容易出错。

现在，这一问题被彻底解决！V11 Update 1 引入了全新的「权限组」模式，让权限管理回归本质：**权限 = 能力 + 范围**。

### 标准化的权限维度，开箱即用

- 系统现在**原生提供清晰的权限维度**，开发者无需再自行设计权限逻辑。
- 你可以直接创建和管理「权限组」，为每个组分配具体的功能权限（如“可编辑”“可导出”），再将权限组与角色关联，实现灵活又规范的权限控制。
- 为保障兼容性，原有的「角色模式」依然保留，老项目可无缝运行。

### 支持前端配置权限

基于权限组模式，我们新增了「角色权限组管理」命令插件。开发者可以利用该插件，在前端页面中**自主构建权限配置模块**，例如：

- 为部门管理员提供权限分配界面
- 让用户自助申请或查看权限范围

真正实现“权限管理”与业务流程的深度融合。

## 企业级精细化管控：支持「分级授权」 {{ type: 'feature', typeText: '权限' }}

依托全新的权限组架构，活字格在 V11 Update 1 中正式支持 「分级授权」 能力。

在服务器管理控制台中，管理员可开启 「分级授权」模式，基于企业组织架构，在应用层面实现多层级、细粒度的权限管控。

典型应用场景：
- **集团统管，部门自治**
  
  集团统一管理平台，但允许 A、B 等子部门独立发布和管理本部门专属应用，彼此数据与应用完全隔离，互不可见。

- **共用应用，分权管理**
  
  A、B 部门员工共同使用同一套应用（如 HR 系统），但仅 A 部门管理员有权配置和管理该应用，B 部门仅能使用，无法修改设置，实现“用”与“管”分离。

这两项更新共同构建了活字格面向中大型企业的新一代权限体系：
- 对开发者：权限设计更标准、更简单，告别“角色模拟”
- 对企业：支持组织架构驱动的精细化授权，满足合规与自治双重需求

## 表格报表插件更名 {{ type: 'business', typeText: '官方付费插件' }}

之前名为「表格报表插件」的功能，自上线以来因其名称与产品中的「报表模块」较为相似，容易让部分用户在商务评估或功能选型时产生困惑。
