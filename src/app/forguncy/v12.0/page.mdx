## 智能辅助，效率倍增 {{ type: 'feature', typeText: '效率' }}

同样的项目，一半的时间；同样的时间，双倍的产出！

### 智能参数匹配

过去，在使用「数据表操作」这样的命令，开发者需要手动匹配字段所对应的参数值。这对于复杂的应用场景开发来说，是一个非常繁琐且耗时的过程。在 V12 中，系统会自动识别上下文参数，无需手动匹配。

### 一键生成标准服务端接口

![智能生成服务端命令](@/images/generate-server-comand.gif)

基于服务端命令的前后端分离开发模式一直是活字格开发中的最佳实践。在 V12 中，新增了一键生成标准服务端接口的功能。基于指定的业务表，系统会自动生成对应的服务端接口，包括新增、修改、删除操作。

### 对象导航，快速检索

现在活字格中任意对象都可以被导航。一键跳转到页面、报表、工作流等对应对象，告别"选择后手动查找"。

插件也支持直接定位到使用位置上，方便评估影响范围。

支持查找行、列和单元格范围的引用，以及删除提示，避免出现 `#REF!` 的错误

在调试阶段，系统支持快捷键 ctrl + M 快速显示当前页面的对象管理窗口。

## 页面设计工程化 {{ type: 'feature', typeText: '页面' }}

活字格的页面能力彻底拥抱工程化，为开发者带来「页面方法」和「页面属性」两大能力。让页面开发从“平铺直叙”变成“结构化编程”。

- **页面方法**：将页面的业务逻辑封装为方法，实现代码的复用与维护。
- **页面属性**：为页面添加自定义属性，方便在运行时动态调整页面行为。开发者将页面状态放置在隐藏单元格中的行为将一去不复返。

![隐藏行列不再是必须的](@/images/remove-hide-row.png)

基于这两大能力，父子页面间的通信也变得更加规范。过去靠"全局变量"或"约定命名单元格"传递数据，耦合混乱，牵一发而动全身。V12 提供了标准的 Get/SetPropertyValue 接口，父页面可以清晰地感知子页面暴露的能力，数据流向一目了然，架构复杂度得到有效控制。

## 工作流支持灵活配置权限 {{ type: 'feature', typeText: '权限' }}

活字格的流程与业务场景是解耦的，流程中的每个节点都需要绑定对应的业务界面。过去，如果需要在流程中对页面字段进行权限控制，就必须在页面上编写大量逻辑判断。V12 引入了「工作流权限」功能，开发者可以直接在流程中配置页面字段的权限，无需再编写额外的页面逻辑。

除此之外，我们还提出了统一流程中心的方案，方便用户在一个地方管理所有的流程。
